# 후크나이츠 (Hook Knights) 설계서 (v2)

> **게임명:** 후크나이츠 / Hook Knights (아크나이츠 패러디)
> **제작사:** 후제 스튜디오 / Hooje Studio (런치 시 인트로 로고 화면 노출)
> 비공개·비수익·친구 전용 멀티버스 배치 디펜스 게임
> 작성일: 2026-06-26 / 상태: **설계 합의 단계** (코딩 전 최종 검토용)
> 본 문서의 수치는 대부분 **튜닝 가능한 제안값**입니다. ★ 표시는 대표님 확인/결정이 필요한 항목.

> **v2 주요 변경:** ① 성장 트랙 = **영웅레벨 + 액티브** 둘만(평타·패시브는 표시 전용·고정). ② **타일 자유화**(전 클래스 어디나 배치, 지상=블록/고지대=논블록). ③ 평타·액티브 **데미지 타입 분리**, Hybrid=둘이 다른 캐릭. ④ 폰트 확정(타이틀 Do Hyeon / 본문 IBM Plex Sans KR / 워드마크 Montserrat Light). ⑤ **튜토리얼 월드0(0-1~0-7)**, 단계별 R 1명 지급. ⑥ R 7캐릭 로스터·스킬·스킨 확정 → §16~18 참조.

## 3. 계정 / 로그인

- **회원가입:** 유저가 직접 ID 정하고 PIN 4자리 설정. ID는 고유.
- **닉네임 (확정):** ID(로그인용, 고유, 비표시) + 닉네임(표시용, 변경가능) 분리.
- **로그인:** ID + PIN. 동일 계정으로 모바일/데스크탑 어디서나 접속, Firestore에서 진행상황 이어받음.
- **신규 계정 시작 상태:** **뽑기권 50장 + 기본 R등급 영웅 1명** 지급. (기본 영웅은 데이터로 지정 → 나중에 교체 가능)
- **관리자:** 계정 삭제 / PIN 리셋 / 재화 지급 가능.

## 4. 클래스 & 데미지 시스템

**클래스 (영문 ID 통일, 화면은 한글 표기)**

| ID | 표기 | 역할 | 기본 블록 수 |
|---|---|---|---|
| `Tank` | 탱커 | 지상 적 막고 버팀 | 2 (패시브로 3·4까지 ↑) |
| `Melee` | 근거리 | 근접 딜러 | 1 |
| `Ranged` | 원거리 | 원거리 딜러 (고지대 배치) | 0 |
| `Support` | 서포터 | 힐/버프/디버프 | 0 |
| `Vanguard` | 뱅가드 | DP 충전 + 전투 겸함 | 1 |

**데미지 타입**

| 타입 | 대응 방어 | 비고 |
|---|---|---|
| `AD` (물리) | Armor(물리방어)에 경감 | |
| `AP` (마법) | MR(마법저항)에 경감 | |
| `Hybrid` (물리복합) | Armor·MR 절반씩 참조 | 단일 방어 특화 적에게 강함 |

- 적 스탯에 **Armor / MR 2종** 존재 → 데미지 타입 선택이 의미를 가짐.
- **블록:** 지상 적은 막혀서 멈춰 싸움. 블록 한도 초과분은 통과. 공중 적은 블록 불가, `Ranged`만 타격.
- **타일 자유화 (v2, 명일방주와 차별점):** 모든 클래스가 **지상·고지대 어디나 배치 가능.** 배치 타일이 블록을 결정 — **지상 = 블록 발생**(탱커 2 / 그 외 1), **고지대 = 블록 0**(지상 적에게 안 맞음). 레인지·서포터도 급하면 지상에 내려 블록 1로 버틸 수 있으나 방어가 낮아 위험. 맵 난이도 조절의 자유도를 줌.

## 5. 전투 시스템

**자원**

- **DP (Deployment Point):** 전역 배치 코스트. 시작값·회복속도 명일방주식. → 제안: 시작 10, +1 DP/초. 뱅가드 스킬로 추가 충전.
- **SP (Skill Point):** 유닛별 스킬 게이지. **시간 기반 충전** (서포터가 가속). 가득 차면 액티브 발동 가능.

**배치 플로우 (명일방주식)**

1. 하단 편성바에서 유닛 선택 → 맵 위로 드래그
2. 확정 전까지 위치 자유 이동, **공격 사거리 타일 하이라이트 표시**
3. 드래그 방향으로 **바라볼 방향** 결정
4. "확정" → 배치 완료 + DP 차감
5. 타일 구분: **지상칸**(블록 발생) / **고지대칸**(블록 0). v2: 클래스 무관 어디나 배치 가능(§4 타일 자유화)

**전투 진행**

- 실시간 자동 전투. 플레이어 개입 = **배치 / 액티브 토글(자동·수동) / 퇴각**.
- **퇴각:** DP 환불 없음, 재배치 쿨다운 후 다시 사용 가능.
- **유닛 사망:** 적 공격으로 HP 0 → 쓰러짐 → 재배치 쿨다운 후 재사용.
- **치명타:** 내 유닛 공격에만 발생(피격엔 없음). 기본 확률 5%, 데미지 ×1.5, **빨간 숫자 + 이펙트**.

**웨이브 / 승패**

- 스테이지 = 여러 웨이브로 적 등장.
- **승리:** 모든 웨이브 처리 + 넥서스 생존.
- **패배:** 넥서스 내구도(HP) 0.

**★ 격자 제안 (세로 화면)**

- 격자 크기: **6열 × 9행** (모바일 세로 비율). 적은 상단에서 진입, 하단 넥서스로 진군.
- 월드1은 **원웨이(단일 경로)**. 이후 월드/스테이지에서 다중 경로·공중 추가.
- 스테이지마다 타일 배치(지상/고지대/이동불가)와 경로를 데이터로 정의.

## 6. 영웅 시스템

**기본 스탯**

| 스탯 | 설명 |
|---|---|
| HP | 체력 |
| ATK | 공격력 |
| Armor | 물리방어 |
| MR | 마법저항 |
| ATK_SPD | 공격속도 (평타 간격) |
| Crit | 치명타 확률 (기본 5%) |
| Block | 블록 수 |
| Cost | 배치 DP 비용 |
| Range | 사거리 (격자 타일 패턴) |
| DMG_TYPE | AD / AP / Hybrid |

- **등급:** R(파랑) / SR(보라) / SSR(금) — 카드 배경색.
- **영웅 레벨:** 무한. 레벨↑ → **HP·공격력·방어·마저 모두 상승**. 필요 EXP 급증, 하지만 무한히 강해짐.
- **성장 방식(확정):** 캐릭터마다 **초기 스탯**이 다르고, **클래스 공통 성장률(%)**이 초기 스탯에 비례 적용. → 초기 공격형은 계속 공격형으로 성장.
- **평타 (v2, 성장 불가):** 레벨업 없음. 성장 탭/상세에 **데미지 타입(AD/AP)과 계수만 표시**. 평타 위력은 영웅레벨(공격력)에 연동.
- **스킬 레벨:** **액티브만** 최대 **30** (성장 가능). 패시브는 고정.

## 7. 스킬 시스템 (v2)

각 영웅은 **평타 + 패시브 1 + 액티브 1**. 성장 탭에는 **레벨 / 평타 / 액티브 / 패시브** 4행이 보이지만, **성장 가능한 건 레벨과 액티브뿐**(평타·패시브는 정보 표시용).

**평타 (고정)** — 데미지 타입+계수만 표시. 위력은 영웅레벨에 연동.

**패시브 (고정)**

- 항상 적용 또는 **조건부 발동**(예: HP 50% 이하 시) 가능.
- **수치 완전 고정** — 레벨업·영웅레벨 비례 모두 없음.
- 파티 버프형 패시브는 **본인이 필드 배치 중일 때만 적용되는 오라**. 같은 종류 버프 중첩 = **합연산**. (마법계열=AP 데미지, 물리계열=AD 데미지 기준)

**액티브 (성장 가능, 최대 30)**

- **SP 가득 → 발동 가능 → 발동 후 다시 충전** (충전시간 = 쿨타임). 별도 쿨타임 없음.
- **자동/수동 토글:** 자동(가득 차면 자동 발동) ↔ 수동(직접 탭) 전투 중 전환.
- 모드 다양: 즉발/지속/장판/버프/유틸/힐/끌어오기 등 (캐릭별 §16).
- 레벨업 시 계수 상승 + **10/20/30에서 효과 진화/추가**.

> 스킬 효과는 데이터 기반 모듈(`skills.json`)로 설계 → 신규 영웅 추가 용이. 7캐릭 실제 스킬은 §16.

## 8. 성장 / 경험치 경제 (핵심)

**재화 = 경험치북 + 뽑기권. 골드·스태미나·크리스탈 없음.**

### 8-1. 경험치북 2종

**범용북 (4색)** — 아무 영웅에게나 사용. 스테이지/출석/우편/이벤트로 획득.

| 색 | EXP(제안) |
|---|---|
| 회색 | 100 |
| 초록 | 500 |
| 빨강 | 2,500 |
| 주황 | 12,500 |

**전용북 (단일 등급, 색=캐릭터 등급)** — 해당 캐릭터에만 사용. 가챠 중복으로 획득.

| 색 | 대상 등급 | EXP(제안) |
|---|---|---|
| 파랑 | R 전용 | 2,000 |
| 보라 | SR 전용 | 2,000 |
| 금색 | SSR 전용 | 2,000 |

- ★ 전용북 EXP는 등급 무관 동일(2,000) 제안. (원하면 등급별 차등 가능)
- **미보유 영웅 해금:** 그 영웅 전용북 **10장** 모으면 해금.

### 8-2. 투입 방식 (확정)

- 성장은 전투 밖 **"성장" 탭**에서만.
- **범용북·전용북 모두** → **영웅레벨 / 액티브** 중 **유저가 골라 투입** (v2: 평타·패시브는 성장 대상 아님).
  - 범용북 = 아무 영웅
  - 전용북 = 해당 영웅만
- 레벨업/스킬업은 책 EXP가 임계치 도달 시 자동 레벨업 (영웅레벨 방식과 동일). 추가 재화 소모 없음.

### 8-3. 레벨 곡선 (제안 — 튜닝 가능)

목표: **스킬 30 찍기 = 영웅 레벨 100 찍기와 비슷한 난이도.** 100 이후로도 무한 성장.

- **영웅 레벨 필요 EXP:** `req(L) = 100 × L²`
  - 예: Lv1→2 = 100, Lv50→51 = 250,000, Lv99→100 = 980,100
  - Lv100 누적 ≈ **32.8M EXP**
- **액티브 레벨 필요 EXP:** `req(L) = 173 × L³`  (v2: 성장 스킬은 액티브 하나)
  - 1→30 누적 ≈ **32.8M EXP** (= 영웅 100 난이도와 매칭) → "30은 매우 빡센" 설계 충족

> 이 두 계수(100, 173)와 지수만 바꾸면 전체 성장 속도를 한 번에 조절할 수 있게 구현합니다.

## 9. 가챠 시스템

**뽑기권 1장 = 1회. 10연차 없음. 크리스탈 없음.**

### 9-1. 드롭 테이블 (확정)

1. **등급 결정:** R 60% / SR 30% / SSR 10%
2. **그 등급 내 랜덤 캐릭터 1명 지정**
3. **결과 타입:**
   - **90% → 그 캐릭터 전용북 1~2장** (1장·2장 확률 동일)
   - **10% → 완전체영웅** (미보유 시 즉시 해금 / 보유 시 전용북 10장)

- 가챠에서 **범용북은 안 나옴.**
- 신규 SSR 직접 획득 ≈ 10%×10% = **1%** (귀하지만 극악 아님). 누적 해금 루트(전용북 10장) 병행.

### 9-2. 천장 (확정)

- 카운터 **50** 도달 → **특정 캐릭터 지정** 선택권.
  - 미보유 → 해금
